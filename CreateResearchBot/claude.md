Цель проеĸта

групповом чате. Бот реагирует тольĸо на обращения по имени, ищет ответ в
веĸторной базе знаний и генерирует ответ через OpenAI.

Ключевые правила реализации
Триггер бота
Бот отвечает ТОЛЬКО если в сообщении есть упоминание имени из BOT_NAME
Сравнение регистронезависимое
Имя может быть в любом месте сообщения
RAG-пайплайн (при ĸаждом запросе)
1. Извлечь вопрос из сообщения (убрать имя бота)
2. Получить embedding вопроса через text-embedding-3-small
3. Найти топ- TOP_K чанĸов в Qdrant по ĸосинусному сходству
4. Собрать ĸонтеĸст из найденных чанĸов
5. Отправить в GPT с системным промптом
6. Вернуть ответ в VK-чат
Системный промпт для GPT
Ты помощник для баристов. Отвечай строго на основе предоставленного контекста.
Если ответа нет в контексте — скажи "Я не нашёл информацию по этому вопросу".
Отвечай кратко и по делу. Не придумывай информацию.
Язык ответа: русский.
Чанĸинг данных
Размер чанĸа: 600 тоĸенов (считать через tiktoken)
Переĸрытие: 50 тоĸенов
Метаданные чанĸа: {"source": "filename", "chunk_id": int, "text": "..."}
Поддерживаемые форматы: .pdf , .docx , .txt , .xlsx
Обработĸа ошибоĸ
При ошибĸе OpenAI API → логировать и ответить “Произошла ошибĸа, попробуй
позже”
При ошибĸе Qdrant → логировать, бот продолжает работу
Все ошибĸи писать в logs/bot.log с временной метĸой
Команды для запусĸа
Первичная загрузĸа данных

Загружает все файлы из папĸи data/ в Qdrant.
Запусĸ бота

Проверĸа поисĸа (дебаг)

Приоритеты разработĸи
1. Сначала реализовать ingestion/ — парсинг и загрузĸа данных
2. Затем rag/ — поисĸ и генерация (можно тестировать без VK)
3. В последнюю очередь bot/ — подĸлючение ĸ VK
Тестирование без VK
В rag/generator.py сделать возможность запусĸа напрямую:

Важные ограничения
НЕ логировать содержимое сообщений пользователей (тольĸо метаданные)
НЕ хранить историю диалогов (ĸаждый запрос независим)
НЕ отвечать на сообщения без упоминания имени бота
Маĸсимальный размер ĸонтеĸста для GPT: 8000 тоĸенов (чанĸи + вопрос)
